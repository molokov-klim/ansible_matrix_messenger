# План по исправлению ошибки с добавлением GPG-ключа репозитория Matrix.org

## Причины ошибки

1. **Устаревший метод добавления GPG-ключа**:
   - Используется устаревший модуль `apt_key` с параметром `apt-key`
   - В новых версиях Ubuntu/Debian `apt-key` больше не рекомендуется к использованию
   - `apt-key` был удален из новых версий Ubuntu/Debian

2. **Неправильный способ добавления ключа**:
   - Ключ добавляется напрямую через `apt-key add`
   - Это может привести к проблемам безопасности и совместимости

## Решения

### Вариант 1: Использовать новый метод добавления GPG-ключа (рекомендуется)

Обновить задачу `Add Matrix.org repository key`, чтобы:
- Скачивать GPG-ключ в файл в `/etc/apt/trusted.gpg.d/`
- Использовать новый метод, рекомендованный для новых версий Ubuntu/Debian

### Вариант 2: Установить ключ через URL

Обновить задачу, чтобы:
- Скачивать ключ напрямую из URL
- Сохранять его в правильное место для новых версий apt

## Рекомендуемое решение

Использовать Вариант 1, так как:
- Это соответствует современным стандартам безопасности
- Совместимо с новыми версиями Ubuntu/Debian
- Рекомендовано разработчиками apt

## План действий

1. Обновить задачу `Add Matrix.org repository key` в `tasks/install_synapse.yml`
2. Заменить использование `apt_key` с `apt-key` на новый метод
3. Скачивать и устанавливать GPG-ключ в `/etc/apt/trusted.gpg.d/`
4. Обновить задачу `Add Matrix.org repository` для использования нового формата
5. Запустить деплой для проверки

## Проверка

После реализации:
- Убедиться, что GPG-ключ успешно устанавливается
- Проверить, что репозиторий Matrix.org добавляется корректно
- Убедиться, что установка Matrix Synapse проходит успешно
